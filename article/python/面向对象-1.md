## 面向过程

- 面向过程:
- 核心是"过程"二字
- 过程的终极奥义就是将程序流程化
- 过程是”流水线"，用来分步骤解决问题的

## 面向对象

- 面向对象:
- 核心是"对象"二字
- 对象是”容器”，用来盛放东西
- 对象的核心就是将"程序"整合

<!--more-->



## 现在要思考用什么方式组织一个容器(对象)？

- **演化一: 字典**
- **缺陷:字典不能包括函数定义的内容**

```python
# 学生的功能
# 传入对象
def stu_info(stu_obj):
    print("name:%s age:%s gender:%s" %(stu_obj['stu_name'],
                                       stu_obj['stu_age'],
                                       stu_obj['stu_gender']
                                       ))


def modify_stu(name,age,gender,stu_obj):
    stu_obj['stu_name'] = name
    stu_obj['stu_age'] = age
    stu_obj['stu_gender'] = gender


stu_obj = {
    'stu_name' : 'charles',
    'stu_age' : 18,
    'stu_gender' : 'male',
    'show_stu_info': stu_info,
    'modify_info':modify_stu
}

stu_info(stu_obj)
```



- **演化二：用系统提供的语法 造出来类**
- **类也是对象 存放相同结构对象共有的数据和功能**

- **面向对象思想整体思路：**
  **1.构思好这个集合需要的数据和功能,一般先用面向过程的思想写出程序。**
  **2.将第一步的 数据 函数 程序 归类 ，再定义类**
  **3.实例化实体**



### 类的定义

```python
'''
1.类的定义
    类名推荐用驼峰体
    类体中除了变量与函数的定义，还可以包含其它任意代码
    注意: 类体代码在定义时就运行了,会产生类的名称空间
    
    在python中，c++的类定义只是作为模板 ,但在python中，类定义自己就是一个类对象。
'''
    
    
#类里面存共有的信息
# 这里假设所有学生的学校都一样
class Student:
    # 1.变量的定义
    school = 'CUMT'
    # 2.功能的定义
    def modify_stu(name, age, gender, stu_obj):
        stu_obj['stu_name'] = name
        stu_obj['stu_age'] = age
        stu_obj['stu_gender'] = gender

# obj.__dict__ 可以查看类对象名称空间里的变量，返回字典。修改这个字典其实就是修改对类对象的信息
# 在python中，c++的类定义只是作为模板 ,但在python中，类定义自己就是一个类对象。
print(Student.__dict__)
'''输出：
{
'__module__': '__main__', 'school': 'CUMT', 'modify_stu': <function Student.modify_stu at 0x03831100>, 
'__dict__': <attribute '__dict__' of 'Student' objects>, 
'__weakref__': <attribute '__weakref__' of 'Student' objects>,'__doc__': None
 }
'''

```



### 类对象的产生（实例化）

```python
# 2.调用类产生对象
# 类的这种调用是生成对象，而不是运行类代码
obA= Student()
print(obA.__dict__)  #输出:{ } 是个空字典，因为对象的名称空间里只存放着对象独有的属性
'''
注意: obj.info = newinfo 的操作相当于 操作字典 obj.__dict__[info] = newinfo
所以，就算类定义里面没有定义 info这个东西 ，这样写也是对的（因为这样操作其实就是在字典添加key），但在C++中,这样写就会报错
'''
obA.name = 'charles'
print(obA.__dict__) #输出：{'name': 'charles'}
```



### 给对象添加特殊属性(\__init\__方法)

```python
# 3.__init__(obj,a,b,c,...)方法
'''
为对象定制自己特有的属性
可以直接按照使用字典的方式来给对象添加修改属性
也可以在类里面定义函数__init__(obj,a,b,c,...)，调用类的时候会自动执行这个函数
并且在调用的时候 会自动传入空对象到obg那个位置，不用自己指定.

'''

class Student:
    # 1.变量的定义
    school = 'CUMT'
    # 2.功能的定义

    # __init__方法 只能返回none，别自己瞎写
    def __init__(stu_obj,name ,age, gender,):
        stu_obj.stu_name = name
        stu_obj.stu_age = age
        stu_obj.stu_gender = gender

    def modify_stu(tu_obj,name, age, gender):
        stu_obj['stu_name'] = name
        stu_obj['stu_age'] = age
        stu_obj['stu_gender'] = gender

    def show_stu(self):
        print(self.stu_name,self.stu_age,self.stu_gender,self.school)



stuA = Student('charles','18','male')
print(stuA.__dict__) # 输出: {'stu_name': 'charles', 'stu_age': '18', 'stu_gender': 'male'}

'''
总结一下实例化发生的事情：
    1.生成一个空对象
    2.调用__init__函数，并自动传入空对象
    3.返回初始化好的对象

'''
```



### 类和对象数据的共享



```python
'''
类存放的是对象共有的数据与功能
类的数据属性是共享给所有对象用的，大家访问的地址都一样

对象的名称空间里只存放着对象独有的属性，而对象们相似的属性是存放于类中的。对象在访问属性时，会优先从对象本身的__dict__中查找，未找到，则去类的__dict__中查找

'''
stuA = Student('charlrs','18','male')
stuB = Student('Amy','18','female')
print(id(stuA.modify_stu),id(stuA.school))
print(id(stuB.modify_stu),id(stuB.school))
'''输出
17296840 53279072
17296840 53279072
'''

print(stuA.school,stuB.school) #输出：CUMT CUMT
Student.school = 'CUMT_BJ'  #这样改会把全部对象的属性都改掉
print(stuA.school,stuB.school) #输出：CUMT_BJ CUMT_BJ
stuA.school = 'ASTY'   #这样改只会这影响stuA的属性
print(stuA.school,stuB.school) #输出：ASTY CUMT_BJ
```

### 绑定

```python
#类中定义的函数是类的函数属性，类可以使用，但必须遵循函数的参数规则，有几个参数需要传几个参数
Student.show_stu(stuA) # charlrs 18 male ASTY
Student.show_stu(stuB) # Amy 18 female CUMT_BJ

# 但其实类中定义的函数主要是给对象使用的，而且是绑定给对象的，虽然所有对象指向的都是相同的功能，但是绑定到不同的对象就是不同的绑定方法，内存地址各不相同

print(id(Student.show_stu)) # 58106040

print(id(stuA.show_stu)) # 20180424
print(id(stuB.show_stu)) # 20180424

# 绑定到对象的方法特殊之处在于，绑定给谁就应该由谁来调用，谁来调用，就会将’谁’本身当做第一个参数自动传入（方法__init__也是一样的道理）

stuA.show_stu()   # 等同于Student.show_stu(stu1)
stuB.show_stu()  # 等同于Student.show_stu(stu2)

# 绑定到不同对象的choose技能，虽然都是选课，但李建刚选的课，不会选给王大力，这正是”绑定“二字的精髓所在。

#注意：绑定到对象方法的这种自动传值的特征，决定了在类中定义的函数都要默认写一个参数self，self可以是任意名字，self必须放第一位，但命名为self是约定俗成的。
```





## 万物皆对象

Python中一切皆为对象，且Python3中类与类型是一个概念，因而绑定方法我们早就接触过的类型list就是类

```python

>>> list
<class 'list'>

#实例化的到3个对象l1,l2,l3
>>> l1=list([1,2,3])
>>> l2=list(['a','b','c'])
>>> l3=list(['x','y'])

#三个对象都有绑定方法append,是相同的功能,但内存地址不同
>>> l1.append
<built-in method append of list object at 0x10b482b48>
>>> l2.append
<built-in method append of list object at 0x10b482b88>
>>> l3.append
<built-in method append of list object at 0x10b482bc8>

#操作绑定方法l1.append(4),就是在往l1添加4,绝对不会将4添加到l2或l3
>>> l1.append(4) #等同于list.append(l1,4)
>>> l1
[1,2,3,4]
>>> l2
['a','b','c']
>>> l3
['x','y']
```
