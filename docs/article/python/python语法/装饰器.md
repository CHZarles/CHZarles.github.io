## 2.装饰器


**什么是装饰器?**
    一种工具,可以定义成函数
    什么叫装饰？为其它事物额外添加的点缀
        =》装饰器通常是: 装饰其他函数，为其它函数添加额外的功能的工具
        
**为何要用装饰器？**
    在不**修改被装饰对象源码**，以及**调用方式**的前提下，为对象添加新功能
    开放封闭原则
        开发：对拓展功能开放
        封闭：对修改源码封闭



## 3.案例推演

**这里有一个原函数**

```python
#源代码
def index(x,y):
    time.sleep(0.1)
    print(x,y);
```



**现在要求在不修改源码的情况下添加一个  “统计函数运行时间” 的功能**

装饰器的要求达到的目的有两个：

- 不修改源码
- 不更改调用方式

 我们不能一步到位 ，但可以按照直觉先把能想到的写出来，后续再做修改



#### 推演1.不修改源码的写法

```python
#做到不修改源代码是很容易的
#这是第一层修饰
import time
def wrapper(*args,**kwargs):
    start = time.time()
    res = index(*args,**kwargs) #管它有没有返回值 写上就没错 
    end = time.time()
    print(end-start)
    return res
```

#### 推演2.不修改原来的调用方法(1)

根据上面的代码 要实现需求的功能 必须调用wrapper(*args,**kwargs)

下面是解决方法

```python
'''
利用闭包和函数对象的思想 
这是第二层修饰,在原来的wrapper外面也套了一层 ，把函数也抽象了
此时这个wrapper_otter(func)就是通用函数了
'''

def wrapper_outter(func):
    def wrapper(*args,**kwargs):
        start = time.time()
        res = func(*args,**kwargs) 
        end = time.time()
        print(end-start)
        return res
    return wrapper

'''
针对上面的index函数
调用步骤为
'''
index = wrapper_outter(index)
index(1,2) 

'''
输出:
1 2
0.10091137886047363
'''

```

#### 推演3 不修改原来的调用方法(2)

为了要实现不修改原来的调用 还是要自己执行一下 

`index = wrapper_outter(index)`

**python提供了语法糖化简了这个步骤**

```python
'''
在定义index函数的时候添加 
@装饰器名称 
'''

@wrapper_outter
def index(x,y):
    time.sleep(0.1)
    print(x,y);
    
'''
调用
'''
index(1,2) 

'''
输出:
1 2
0.10091137886047363
'''

```

#### 推演4 添加多个装饰器

**现在在原有的基础上添加一个作者认证功能**

要额外写一个装饰器

```python
'''
作者认证的通用装饰器
输出作者 charles
'''
def author(func):
    def wrapper(*args,**kwargs):
        res = func(*args,**kwargs) 
        print("charles")
        return res
    return wrapper

'''
定义时另起一行添加语法糖
'''
@wrapper_outter
@author
def index(x,y):
    time.sleep(0.1)
    print(x,y);
    
    
'''
输出:
1 2
charles
0.10034942626953125

语法执行顺序是
index = wrapper_outter(author(index))  
index(1,2)
所以这里先输出 时间 再输出 作者
'''
```

#### 推演5  设计有参数的装饰器

总结一下，上面无参装饰器的模板

```python
#无参装饰器模板
def com_outter(func):
    def wrapper(*args, **kwargs):
        #实现装饰功能的代码
        res = func(*args, **kwargs)
        #实现装饰器功能代码
        return res

```



下面一个案例看一下有参装饰器

```python
#有参装饰器模板demo
#先写两个独立的函数
def save_to_database():
    pass

def save_to_Memo():
    pass

#想给装饰器添加函数的直观写法
def saver(func,op):
    def wrapper(*args, **kwargs):
        #实现装饰功能的代码
        if op == 1:
            print("保存到数据库")
            res = func()
        elif op == 2:
            print("保存到磁盘")
            res = ()
        #实现装饰器功能代码
        return res
    return wrapper



#显然上面的写法不通用 也不方便调用装饰器 没有达到装饰器"润物细无声"的功效
# 利用闭包的思想将saver()封闭，写一个闭包函数，再利用语法糖预处理
# 定义步骤

def saver_outter(op):
    def saver(func):
        ifunc = func
        def wrapper(*args, **kwargs):
            #实现装饰功能的代码
            if op == 1:
                print("保存到数据库")
            elif op == 2:
                print("保存到磁盘")
            res = ifunc(*args, **kwargs)
            #实现装饰器功能代码
            return res
        return wrapper
    return saver


'''
 语法糖最本质就是  @函数地址 然后再将定义的函数传参到这个地址
 @saver_outter(1) 会产生这样的效果
    1.系统会先运行 tmp = saver_outter(1)，
    2.语句替换为@tmp 
 由此可见这里 @的是一个的返回值 , 即saver 
'''
@saver_outter(1)
def save_to_database():
    pass

@saver_outter(2)
def save_to_Memo():
    pass

#这样就神不知鬼不觉的把save_to_database给装饰了
#调用demo
save_to_database()#输出 保存到数据库
```
