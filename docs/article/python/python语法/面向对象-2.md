## 隐藏数据属性

- Python的Class机制采用双下划线开头的方式将属性隐藏起来（设置成私有的）
- 但其实这仅仅只是一种变形操作，类中所有双下滑线开头的属性都会在类定义阶段.(只有开头有双下划线 不是前后要双下划线 __xxx__是内置属性的意思)
- 检测语法时自动变成“_类名__属性名”的形式：

```python
class Foo:
    __N=0 # 变形为_Foo__N

    def __init__(self): # 定义函数时，会检测函数语法，所以__开头的属性也会变形
        self.__x=10 # 变形为self._Foo__x

    def __f1(self): # 变形为_Foo__f1
        print('__f1 run')

    def f2(self):  # 定义函数时，会检测函数语法，所以__开头的属性也会变形
        self.__f1() #变形为self._Foo__f1()

print(Foo.__N) 
# 报错AttributeError:类Foo没有属性__N

obj = Foo()
# print(obbj.__x) # 报错AttributeError:对象obj没有属性__x
```

<!--more-->

> 这种变形需要注意的问题是：
>
> 1.在类外部无法直接访问双下滑线开头的属性，但知道了类名和属性名就可以拼出名字：_类名__属性，
> 然后就可以访问了，如Foo._A__N，所以说这种操作并没有严格意义上地限制外部访问，
> 仅仅只是一种语法意义上的变形。

```python
print(Foo.__dict__)
'''
{'__module__': '__main__', '_Foo__N': 0, '__init__': <function Foo.__init__ at 0x02B74C88>, '_Foo__f1': <function Foo.__f1 at 0x02B01148>, 'f2': <function Foo.f2 at 0x02B01100>, '__dict__': <attribute '__dict__' of 'Foo' objects>, '__weakref__': <attribute '__weakref__' of 'Foo' objects>, '__doc__': None}
'''
print(obj.__dict__) #{'_Foo__x': 10}
print(Foo._Foo__N #输出:0
print(obj._Foo__x) #输出: 10
print(obj._Foo__N) #输出: 0
```



> 2、隐藏对外不对内:在类内部是可以直接访问双下滑线开头的属性的，比如self.__f1()，因为在类定义阶段类内部双下滑线开头的属性统一发生了变形。

```python
print(obj.f2()) #输出:none
```

> 3、变形操作只在类定义阶段发生一次,在类定义之后的赋值操作，不会变形。

```python
Foo.__M=100
print(Foo.__dict__)
'''
输出:{'....省略.... ,'__M': 100}
'''
```

## 装饰器property

**装饰器是在不修改被装饰对象源代码以及调用方式的前提下为被装饰对象添加新功能的可调用对象**

> **property是一个装饰器，是用来绑定给对象的方法伪造成一 个数据属性**

#### 案例一

```python
#property 是一个装饰器
'''
成人的BMI数值:
过轻:低于18.5
正常: 18.5-23.9
过重: 24-27
肥胖: 28-32
非常肥胖，高于32
体质指数(BMI) =体重(kg) +身高^2 (m)
EX: 70kg+ (1. 75x1.75) =22.86
'''

class people:
    def __init__(self,name,weight,height):
        self.name = name
        self.weight = weight
        self.height = height

    @property
    def bmi(self):
        return self.weight/(self.height**2)


p = people('charles',62,1.75)
print(p.bmi)  #伪装成数据 ,触发方法bmi的执行，将obj自动传给self，执行后返回值作为本次引用的结果
```

#### 案例二

```python
#案例2
#隐藏,开接口
class People:
    def __init__(self,name):
        self.__name = name


    def get_name(self):
        return self.__name

    def set_name(self,val):
        if type(val) is not str:
            print("you should use str as argv")
            return
        self.__name = val

    def del_name(self):
        print("不让删除")


    #魔术般的操作 把函数接口伪装成数据属性的样子
    name = property(get_name,set_name,del_name)


pe = People('Hank')
pe.name = 'New hank'
del pe.name #输出:不能删除
print(pe.name)#输出：New hank
```

#### 案例三

```python
#案例三
#python3+的实现案例二功能的方法
#首先把相关函数改成一样的名字，再加装饰器
class PeoPle:
    def __init__(self,name):
        self.__name = name

    @property    #name = proper(name) , 这样设置obj.name调用
    def name(self):
        return self.__name

    @name.setter  #这样设置obj.name = 'xxx'调用
    def name(self,val):
        if type(val) is not str:
            print("you should use str as argv")
            return
        self.__name = val

    @name.deleter  #这样设置del obj.name 调用
    def del_name(self):
        print("不让删除")


pe = PeoPle('charles')
pe.name = 'New charles' #触发name.setter装饰器对应的函数name(f,’Egon')

print(pe.name)
```
