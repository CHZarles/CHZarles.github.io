## 树和图的存储

**因为树也是一种特殊的图，是无环连通图，所以我们只要记图的存储方式就好。**

**图又分有向图和无向图，因为无向图是特殊的有向图，所以只要记有向图的存储方式就好。**



那么我们来看看有向图的存法：

![image.png](https://gitee.com/chzarles/images/raw/master/imgs/006eb5E0gy1g8bqhcq8j7j30i50akaao.jpg)

### 邻接矩阵

设图 G(V,E) 的顶点标号为0,1,…，N-1 , 那么可以令二维数组` G[N][N]` 的两维分别表示图的顶点标号，
<!--more-->

- 如果 `G[i][j]==1` , 则说明顶点i和顶点j之间有边;
- 如果`G[i][j]==0`,则说明讲顶点1和顶点j之间不存在边
- 如果存在边权，则可以令`G[i][j]`存放边权，对不存在的边可以设边权为0、-1或是一个很大的数。

(样例略)

**注意，如果题目的点很多，开邻接矩阵，内存可能会超限。（数组大小 n^2*sizeof(数据类型) ） 。**



#### 参考代码

```c++
//初始化 ，n是点数 ， m是边数
const int N = XXX;
int d[N][N];
for(int i = 1 ; i<=n ;i++)
    for(int j =1;j<=n ;j++)
        if(i == j) d[i][j]=0;
        else d[i][j]=inf;

//存边
while(m--)
    {
        int a,b,c;
        cin>>a>>b>>c;
        d[a][b]=min(d[a][b],c);
        
    }
```



### 邻接表

设图G(V,E) 的顶点编号为0,1…，N-1, **每个顶点都可能有若干条出边**，如果把同一个顶点的所有出边放在一个列表中，那么N个顶点就会有N个列表(没有出边，则对应空表)这N个列表被称为图G的邻接表，记为`Adj[N]`,其中`Adj[i]` 存放 顶点i 的所有出边组成的一个列表。**（ 由于列表一般可以用链表表示，所以邻接表的写法和列表的写法很像）**

```c++
//初始化
const int N = XXX,M = 2*N;
int h[N],e[N],ne[M],idx=0;
//这句话放进main()
memset(h,-1,sizeof h);

//加边
void add(int a,int b)
{
    e[idx] = b , ne[idx] = h[a] , h[a] = idx++;
}
```



## 树和图的深搜

***如无特殊要求，有向图和无向图是深搜代码基本是不区分的。***

***参考代码:***

```c++
bool st[N] ; //st[i]=true 意思是编号为i的点走过了
void dfs(int u)
{	
    st[u] = true;
    //遍历链表
    for(int i = h[u]; i != -1; i ++)
    {
		int j = e[i];
        if(!st[j]) 
            dfs(j);
    }
}
```
